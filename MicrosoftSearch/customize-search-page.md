# <a name="customize-the-microsoft-search-page"></a>Настройка страницы поиска Microsoft

Создавайте вертикальные и типы результатов поиска, вы можете настроить результаты поиска, которые будут отображаться для пользователей при поиске в **SharePoint**, **Office.com** и **Microsoft Search в Bing** . Вертикальные черты облегчают пользователям поиск сведений, которые у них есть разрешение на просмотр.  Например, вы можете создать вертикаль поиска для данных маркетингового анализа от стороннего программного обеспечения для пользователей в отделе маркетинга. Вы также можете определить типы результатов и настроить макет для этих данных.  

На этих уровнях можно создавать вертикальные и типы результатов: 

- Уровень организации — при добавлении вертикальной оси на уровне Организации он отображается на странице результатов поиска при выполнении поиска пользователями на начальной странице SharePoint, на Office.com и на Bing.com. 
- Уровень сайта — например, вам может потребоваться разрешить сотрудникам службы клиента искать инциденты "серьезность 1" непосредственно на сайте SharePoint своего отдела. 

## <a name="whats-a-search-vertical"></a>Что такое вертикальный поиск?

В верхней части страницы результатов поиска Microsoft Search есть строка вкладок, они являются вертикалями поиска. В вертикальном поиске отображаются только результаты определенного типа или из определенного содержимого, например только файлы или Новости. По умолчанию в Microsoft Search отображаются все вертикальные, люди, файлы, сайты и новости.  

Вы можете добавить вертикальные вертикали поиска, которые относятся к вашей организации. Они будут отображаться на странице результатов Microsoft Search в SharePoint, Office.com и Bing.  Например, вы можете создать один вертикальный вертикаль для контента, связанного с маркетингом, и другой для продажи в зависимости от типа данных, которые потребуются для каждой группы. Можно добавить вертикальные значения, чтобы показать результаты только из индексированного контента с помощью соединителей.  

**Заявление об отказе:** Вертикальные и типы результатов в настоящее время находятся в режиме предварительного просмотра в рамках предварительного просмотра Microsoft Connectors. Невозможно создать вертикальную вертикальную информацию для контента, расположенного в SharePoint или на основе контента, индексируемого соединителем общей папки. Дополнительные сведения о предварительной версии можно найти в разделе [Предварительный просмотр соединителей](connectors-preview.md). Чтобы принять участие в предварительной версии, необходимо сначала добавить [форму подписки на предварительный просмотр соединителей Microsoft Graph](https://forms.office.com/Pages/ResponsePage.aspx?id=v4j5cvGGr0GRqy180BHbRxWYgu82J_RFnMMATAS6_chUNVYwNU1CMDNZUDBSSDZKWVo2RDJDRjRLQi4u).

## <a name="things-to-consider"></a>Факторы, которые необходимо учитывать...

Прежде чем начать, убедитесь, что соединитель был проиндексирован. В зависимости от размера файла может потребоваться до 48 часов.

Вы не можете создать вертикальный контент для контента в SharePoint или из контента, индексируемого соединителем общей папки. Узнайте, как индексировать контент (ссылка на документацию по соединителю).

На высоком уровне существует три основных действия по добавлению вертикальной. 

1. Создание вертикальной оси — на этом этапе определяются имя и источник контента, а также область поиска контента, в котором выполняется поиск. 
2. Определите, как будут выглядеть результаты для этого вертикального вертикального выравнивания.  
3. Включите вертикальное (для отображения) на странице вертикальный список.   

## <a name="step-1-create-the-search-vertical"></a>Шаг 1: создание вертикали поиска

После запуска мастера вы узнаете, как определить имя, источник контента и область контента, в котором будет выполняться поиск по вертикали. Вертикальная линия будет создана в отключенном состоянии. Вертикальная настройка будет включена позже.

Можно использовать ограниченный набор [языка запросов ключевых слов (KQL)](https://docs.microsoft.com/sharepoint/dev/general-development/keyword-query-language-kql-syntax-reference) , чтобы сузить область, а на странице — список доступных свойств. Мы рекомендуем использовать бесплатные текстовые ключевые слова и ограничения свойств с помощью логических операторов для создания KQL 

### <a name="create-a-vertical-at-the-organization-level"></a>Создание вертикальной оси на уровне Организации

Чтобы создать вертикальный элемент в Microsoft Search в SharePoint Home, Bing или Office.com, выполните указанные ниже действия.

1. В центре администрирования Microsoft 365 перейдите к разделу **Параметры** > > **вертикали** **поиска Microsoft**.
1. Нажмите кнопку **Добавить** , чтобы приступить к работе.  

### <a name="create-a-vertical-at-the-site-level"></a>Создание вертикальной вертикальной версии на уровне сайта

1. На сайте SharePoint, на котором необходимо создать вертикальный текст, перейдите к разделу **Параметры**.
1. Выберите **сведения о сайте**, а затем **Просмотрите все параметры сайта**.
1. Найдите раздел **Microsoft Search** , а затем выберите **Настройка службы поиска Майкрософт для этого семейства веб-сайтов**.
1. В области навигации выберите **Пользовательский интерфейс**, а затем перейдите на вкладку **вертикальные черты** .
1. Чтобы добавить вертикальный элемент, нажмите кнопку **Добавить**. <br>
ИЛИ <br>Чтобы изменить вертикальное расположение, выберите его в списке.

Помните, что вертикальные повертикали создаются в отключенном состоянии. Вам по-прежнему потребуется включить их, прежде чем пользователи смогут увидеть вертикальные черты.

## <a name="step-2-create-the-result-types"></a>Шаг 2: создание типов результатов

Вы можете определить, как результаты отображаются в вертикальном порядке, разрабатывая макет с помощью типов результатов. Макет результатов позволяет показывать важную информацию непосредственно в результатах поиска, чтобы пользователи не могли щелкать каждый результат, чтобы узнать, нашли ли они то, что они искали.

Тип результата поиска  это правило, благодаря которому результаты поиска разных типов отображаются различными способами. Оно состоит из следующих компонентов.

- *Одно или несколько условий* для сравнения результатов поиска, например источник контента результатов поиска.  
- *Структура результатов* , используемая для результатов поиска, которые удовлетворяют условиям. Структура результатов управляет способом отображения всех результатов, удовлетворяющих условиям, и их поведения на странице результатов поиска.

**Необходимо создать по крайней мере один тип результата для результатов, отображаемых по вертикали.** Можно создать несколько типов результатов для каждого вертикального элемента, что позволяет использовать разные макеты для разных типов результатов. Например, вы можете настроить инциденты "серьезность 1" для более заметных цветов и большего шрифта в сравнении с происшествиями "серьезность 3". 

 После запуска мастера необходимо будет выполнить действия по определению имени, источника контента и условий для типа результатов. С помощью представления списка можно определить приоритет типа результата. 
  
### <a name="create-a-result-type-at-the-organization-level"></a>Создание типа результатов на уровне Организации

1. В центре администрирования Microsoft 365 перейдите к разделу **Настройка** > **службы поиска Microsoft**и выберите **Тип результата**.
1. Чтобы добавить **Тип результата**, нажмите кнопку **Добавить**. Чтобы изменить тип результата, выберите тип результата в соответствующем списке.
 
### <a name="create-a-results-type-at-the-site-level"></a>Создание типа результатов на уровне сайта

1. На сайте SharePoint, на котором вы хотите создать тип результата, перейдите к разделу **Параметры**.
1. Выберите **сведения о сайте**, а затем **Просмотрите все параметры сайта**. 
1. Найдите раздел Microsoft Search, а затем выберите **Настройка службы поиска Майкрософт для этого семейства веб-сайтов**.
1. В области навигации перейдите к **элементу пользовательский интерфейс**, а затем выберите вкладку Тип результата.
1. Чтобы добавить тип результата, нажмите кнопку **Добавить**. <br> ИЛИ <br>Чтобы изменить тип результата, выберите тип результата в списке.

### <a name="view-the-vertical-after-enabling"></a>Просмотр вертикальной оси после включения

После включения вертикальной установки может потребоваться некоторое время, прежде чем вы сможете просмотреть его.
Если вы хотите дождаться его включения, можно добавить *качеклеар = true* в URL-адрес в SharePoint и Office.com для немедленного просмотра по вертикали.

Ниже перечислены распространенные проблемы, которые могут возникнуть, и действия по их устранению.


|Error  |Action  |
|---------|---------|
|Отображается ошибка "что-то пошло не так" на вертикальном|   Для завершения настройки необходимы как вертикальные, так и типы результатов. Убедитесь, что вы создали оба для одного источника контента.      |
|Почему я не вижу результатов, несмотря на то, что он создан? | Использование типов результатов занимает несколько минут, так как эти параметры, как правило, кэшируются. Подождите несколько минут и повторите попытку.        |
|На странице "вертикальный текст" или "тип результата" не отображаются источники контента     |      Убедитесь, что вы настроили соединители и индексированные данные   |
