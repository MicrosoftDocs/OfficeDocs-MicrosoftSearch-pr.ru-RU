# <a name="customize-the-search-results-page"></a>Настройка страницы результатов поиска

Вы можете создавать вертикальные и типы результатов поиска, чтобы настроить результаты поиска, которые пользователи видят при поиске в Microsoft [SharePoint](https://sharepoint.com/), [Microsoft Office](https://office.com)и Microsoft Search в [Bing](https://bing.com). Вертикальные черты облегчают пользователям поиск сведений, которые у них есть разрешение на просмотр. Например, вы можете создать вертикаль поиска для данных маркетингового анализа от стороннего программного обеспечения для пользователей в отделе маркетинга. Вы также можете определить типы результатов и настроить макет для этих данных.  

На этих уровнях можно создавать вертикальные и типы результатов:

- **Уровень организации** — при добавлении вертикальной оси на уровне Организации он отображается на странице результатов поиска при выполнении пользователями поиска на начальной странице [SharePoint](https://sharepoint.com/) , на [Office](https://office.com)или в [Bing](https://bing.com).
- **Уровень сайта** — например, вы можете предоставить сотрудникам службы клиентов возможность искать инциденты *серьезности 1* непосредственно на сайте SharePoint своего отдела.

## <a name="search-verticals-explained"></a>Описание вертикального поиска

В верхней части страницы результатов поиска Microsoft Search есть строка вкладок. Ниже приведены вертикальные результаты поиска. В вертикальном поиске отображаются только результаты определенного типа или определенного содержимого. Примерами могут быть **файлы** или **Новости**. По умолчанию в Microsoft Search отображаются **все** вертикальные, **люди** , **файлы** , **сайты** и **Новости**.  

Вы можете добавить вертикальные вертикали поиска, которые относятся к вашей организации. Они будут отображаться на странице результатов поиска Microsoft в [SharePoint](https://sharepoint.com/), [Office](https://Office.com)и [Bing](https://bing.com). Например, вы можете создать вертикальный контент для контента, связанного с маркетингом, и другой для продажи в зависимости от типа сведений, необходимых для каждой группы. Можно добавить вертикальные значения, чтобы показать результаты только из индексированного контента с помощью соединителей.  

>[!NOTE]
> Вертикальные и типы результатов в настоящее время находятся в режиме предварительного просмотра в рамках предварительного просмотра соединителей Microsoft Graph и не могут использоваться для контента, находящегося в [SharePoint](https://sharepoint.com/). Дополнительные сведения о предварительной версии см в разделе [Предварительный просмотр соединителей](connectors-preview.md). Чтобы принять участие в предварительной версии, необходимо сначала добавить [форму подписки на предварительный просмотр соединителей Microsoft Graph](https://forms.office.com/Pages/ResponsePage.aspx?id=v4j5cvGGr0GRqy180BHbRxWYgu82J_RFnMMATAS6_chUNVYwNU1CMDNZUDBSSDZKWVo2RDJDRjRLQi4u).

## <a name="things-to-consider"></a>Важные факторы

Прежде чем начать, убедитесь, что соединитель индексирован. Это может занять до 48 часов, в зависимости от размера файла.

Добавление вертикальной вертикальной оси состоит из трех основных этапов:

1. Создайте вертикальную. На этом шаге задаются имя, источник контента и область поиска контента для поиска.
2. Определите, как будут выглядеть результаты для этого вертикального вертикального выравнивания.  
3. Включите вертикальное (для отображения) на странице вертикальный список.

## <a name="step-1-create-the-search-vertical"></a>Шаг 1: создание вертикали поиска

После запуска мастера вы узнаете, как определить имя и источник контента, а также область поиска контента для поиска. Вертикальная линия будет создана в отключенном состоянии. Его можно будет включить позже.

Для сужения области можно использовать ограниченный набор [ключевых слов языка запросов (KQL)](https://docs.microsoft.com/sharepoint/dev/general-development/keyword-query-language-kql-syntax-reference) . На этой странице перечислены доступные свойства. Для создания KQL рекомендуется использовать ключевые слова и ограничения свойств FREETEXT с логическими операторами.

### <a name="create-a-vertical-at-the-organization-level"></a>Создание вертикальной оси на уровне Организации

Чтобы создать вертикальную область поиска в Microsoft Search [в домашнем](https://sharepoint.com/) , [Office](https://office.com)или [Bing](https://bing.com), выполните указанные ниже действия.

1. В  [центре администрирования Microsoft 365](https://admin.microsoft.com)перейдите на страницу  [**вертикальные черты**](https://admin.microsoft.com/Adminportal/Home#/MicrosoftSearch/verticals).
1. Нажмите кнопку **Добавить** , чтобы приступить к работе.  

### <a name="create-a-vertical-at-the-site-level"></a>Создание вертикальной вертикальной версии на уровне сайта

1. На сайте [SharePoint](https://sharepoint.com/) , на котором вы хотите вертикально, перейдите к разделу **Параметры**.
1. Выберите **сведения о сайте** , а затем **Просмотрите все параметры сайта**.
1. Найдите раздел **Microsoft Search** , а затем выберите **Настройка службы поиска Майкрософт для этого семейства веб-сайтов**.
1. В области навигации выберите  **Пользовательский интерфейс** , а затем перейдите на вкладку **вертикальные черты** .
1. Чтобы добавить вертикальный элемент, нажмите кнопку **Добавить**.
  Чтобы изменить вертикальное расположение, выберите его в списке.

Помните, что вертикальные повертикали создаются в отключенном состоянии. Их необходимо включить, прежде чем пользователи смогут их видеть.

## <a name="step-2-create-the-result-types"></a>Шаг 2: создание типов результатов

Вы можете определить, как результаты отображаются в вертикальном порядке, разрабатывая макет с помощью типов результатов. Макет результатов позволяет показывать важную информацию непосредственно в результатах поиска, поэтому пользователям не нужно выбирать каждый результат, чтобы узнать, нашли ли они то, что они искали.

Тип результата поиска  это правило, благодаря которому результаты поиска разных типов отображаются различными способами. Оно состоит из следующих компонентов.

- **Одно или несколько условий** для сравнения результатов поиска, например источник контента результатов поиска.  
- **Структура результатов** , используемая для результатов поиска, которые удовлетворяют условиям. Структура результатов управляет тем, как все результаты, удовлетворяющие условиям, отображаются и ведут себя на странице результатов поиска.

**Необходимо создать по крайней мере один тип результата для результатов, отображаемых по вертикали.** Можно создать несколько типов результатов для каждого вертикального, что позволяет использовать различные макеты для разных типов результатов. Например, вы можете настроить инциденты *серьезности 1* для более заметных цветов и большего шрифта в сравнении с происшествиями *уровня серьезности 3* .

После запуска мастера вы узнаете, как определить имя, источник контента и условия для типа результатов. С помощью представления списка можно определить приоритет типа результата.
  
### <a name="create-a-result-type-at-the-organization-level"></a>Создание типа результатов на уровне Организации

1. В [центре администрирования Microsoft 365](https://admin.microsoft.com)перейдите к разделу [**типы результатов**](https://admin.microsoft.com/Adminportal/Home#/MicrosoftSearch/resulttypes).
1. Чтобы добавить **Тип результата** , нажмите кнопку  **Добавить**. Чтобы изменить тип результата, выберите тип результата в соответствующем списке.

### <a name="create-a-results-type-at-the-site-level"></a>Создание типа результатов на уровне сайта

1. На сайте [SharePoint](https://sharepoint.com/) , на котором вы хотите создать тип результата, перейдите к разделу **Параметры**.
1. Выберите **сведения о сайте** , а затем **Просмотрите все параметры сайта**.
1. Найдите раздел Microsoft Search, а затем выберите **Настройка службы поиска Майкрософт для этого семейства веб-сайтов**.
1. В области навигации перейдите в раздел  **Настраиваемый интерфейс** и выберите вкладку **Тип результата** .
1. Чтобы добавить тип результата, нажмите кнопку **Добавить**.  Чтобы изменить тип результата, выберите тип результата в списке.

### <a name="view-the-vertical-after-its-enabled"></a>Просмотр вертикального элемента после его включения

После включения вертикальной установки может потребоваться некоторое время, прежде чем вы сможете просмотреть его. Если вы не хотите ждать после включения, вы можете добавить **качеклеар = true** в URL-адрес в [SharePoint](https://sharepoint.com/) и [Office](https://office.com) для немедленного просмотра по вертикали.

## <a name="troubleshooting"></a>Устранение неполадок

Ниже перечислены распространенные проблемы, которые могут возникнуть, а также действия по их устранению.

|Error  |Действие  |
|---------|---------|
| Отображается сообщение об ошибке "что-то пошло не так" на вертикальном. | Для завершения настройки необходимы как вертикальные, так и типы результатов. Убедитесь, что вы создали оба для одного источника контента. |
| Я не вижу моего макета результатов, несмотря на то, что он был создан. | Это занимает несколько минут, так как эти параметры, как правило, кэшируются. Подождите несколько минут и повторите попытку.        |
| На странице "вертикальный текст" или "тип результата" не отображаются источники контента. | Убедитесь, что вы настроили соединители и индексированные данные.   |
