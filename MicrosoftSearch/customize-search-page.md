# <a name="customize-the-search-results-page"></a>Настройка страницы результатов поиска

Создавая вертикальные и типы результатов поиска, вы можете настроить результаты поиска, которые будут отображаться для пользователей при поиске в [SharePoint](http://sharepoint.com/), [Microsoft Office](https://Office.com)и Microsoft Search в [Bing](https://Bing.com). Вертикальные черты облегчают пользователям поиск сведений, которые у них есть разрешение на просмотр. Например, вы можете создать вертикаль поиска для данных маркетингового анализа от стороннего программного обеспечения для пользователей в отделе маркетинга. Вы также можете определить типы результатов и настроить макет для этих данных.  

На этих уровнях можно создавать вертикальные и типы результатов:

- **Уровень организации** — при добавлении вертикальной оси на уровне Организации она отображается на странице результатов поиска при выполнении пользователями поиска на начальной странице [SharePoint](http://sharepoint.com/) , в [Office](https://Office.com)и [Bing](https://Bing.com).
- **Уровень сайта** — например, вам может потребоваться разрешить сотрудникам службы клиентов искать инциденты **серьезности 1** непосредственно на сайте SharePoint своего отдела.

## <a name="search-verticals-explained"></a>Описание вертикального поиска

В верхней части страницы результатов поиска Microsoft находится строка вкладок, которые являются вертикалями поиска. В вертикальном поиске отображаются только результаты определенного типа или определенного содержимого. Пример: только файлы или Новости. По умолчанию в Microsoft Search отображаются **все**вертикальные, **люди**, **файлы**, **сайты**и **Новости**.  

Вы можете добавить вертикальные вертикали поиска, которые относятся к вашей организации. Они будут отображаться на странице результатов поиска Microsoft в [SharePoint](http://sharepoint.com/), [Office](https://Office.com)и [Bing](https://Bing.com). Например, вы можете создать один вертикальный контент для контента, связанного с маркетингом, а другой для продажи, в зависимости от типа данных, которые потребуются для каждой группы. Можно добавить вертикальные значения, чтобы показать результаты только из индексированного контента с помощью соединителей.  

**Заявление об отказе:** Вертикальные и типы результатов в настоящее время доступны для предварительного просмотра в рамках предварительного просмотра соединителей Microsoft Graph. Невозможно создать вертикальную вертикальную информацию для контента, расположенного в [SharePoint](http://sharepoint.com/) или на основе контента, индексируемого [соединителем файлового ресурса](file-share-connector.md). Дополнительные сведения о предварительной версии можно найти в разделе [Предварительный просмотр соединителей](connectors-preview.md). Чтобы принять участие в предварительной версии, необходимо сначала добавить [форму подписки на предварительный просмотр соединителей Microsoft Graph](https://forms.office.com/Pages/ResponsePage.aspx?id=v4j5cvGGr0GRqy180BHbRxWYgu82J_RFnMMATAS6_chUNVYwNU1CMDNZUDBSSDZKWVo2RDJDRjRLQi4u).

## <a name="things-to-consider"></a>Важные факторы

Прежде чем начать, убедитесь, что соединитель был проиндексирован. В зависимости от размера файла может потребоваться до 48 часов.

Невозможно создать вертикальную вертикальную информацию для контента, расположенного в [SharePoint](http://sharepoint.com/) или на основе контента, индексируемого [соединителем файлового ресурса](file-share-connector.md). Узнайте, как [индексировать контент](configure-connector.md).

На высоком уровне существует три основных действия по добавлению вертикальной оси:

1. Создайте вертикальную. На этом шаге задаются имя, источник контента и область поиска контента для поиска.
2. Определите, как будут выглядеть результаты для этого вертикального вертикального выравнивания.  
3. Включите вертикальное (для отображения) на странице вертикальный список.

## <a name="step-1-create-the-search-vertical"></a>Шаг 1: создание вертикали поиска

После запуска мастера вы узнаете, как определить имя и источник контента, а также область контента для поиска. Вертикальная линия будет создана в отключенном состоянии. Вертикальная настройка будет включена позже.

Можно использовать ограниченный набор [языка запросов ключевых слов (KQL)](https://docs.microsoft.com/sharepoint/dev/general-development/keyword-query-language-kql-syntax-reference) , чтобы сузить область, а на странице — список доступных свойств. Мы рекомендуем использовать бесплатные текстовые ключевые слова и ограничения свойств с помощью логических операторов для создания KQL.

### <a name="create-a-vertical-at-the-organization-level"></a>Создание вертикальной оси на уровне Организации

Чтобы создать вертикальную область поиска в Microsoft Search [в домашнем](http://sharepoint.com/) , [Office](https://Office.com)или [Bing](https://Bing.com), выполните указанные ниже действия.

1. В [центре администрирования](https://admin.microsoft.com)Microsoft 365 откройте раздел **Параметры** > > **вертикали** **поиска Microsoft**.
1. Нажмите кнопку **Добавить** , чтобы приступить к работе.  

### <a name="create-a-vertical-at-the-site-level"></a>Создание вертикальной вертикальной версии на уровне сайта

1. На сайте [SharePoint](http://sharepoint.com/) , на котором необходимо создать вертикальный текст, перейдите к разделу **Параметры**.
1. Выберите **сведения о сайте** , а затем **Просмотрите все параметры сайта**.
1. Найдите раздел **Microsoft Search** , а затем выберите **Настройка службы поиска Майкрософт для этого семейства веб-сайтов**.
1. В области навигации выберите **Пользовательский интерфейс**, а затем перейдите на вкладку **вертикальные черты** .
1. Чтобы добавить вертикальный элемент, нажмите кнопку **Добавить**.  
ИЛИ  
Чтобы изменить вертикальное расположение, выберите его в списке.

Помните, что вертикальные повертикали создаются в отключенном состоянии. По-прежнему необходимо включить их, прежде чем пользователи смогут увидеть вертикальные черты.

## <a name="step-2-create-the-result-types"></a>Шаг 2: создание типов результатов

Вы можете определить, как результаты отображаются в вертикальном порядке, разрабатывая макет с помощью типов результатов. Макет результатов позволяет показывать важную информацию непосредственно в результатах поиска, поэтому пользователям не нужно выбирать каждый результат, чтобы узнать, были ли они найдены.

Тип результата поиска  это правило, благодаря которому результаты поиска разных типов отображаются различными способами. Оно состоит из следующих компонентов.

- **Одно или несколько условий** для сравнения результатов поиска, например источник контента результатов поиска.  
- **Структура результатов** , используемая для результатов поиска, которые удовлетворяют условиям. Структура результатов управляет способом отображения всех результатов, удовлетворяющих условиям, и их поведения на странице результатов поиска.

**Необходимо создать по крайней мере один тип результата для результатов, отображаемых по вертикали.** Можно создать несколько типов результатов для каждого вертикального, что позволяет использовать различные макеты для разных типов результатов. Например, вы можете настроить инциденты **серьезности 1** для более заметных цветов и большего шрифта в сравнении с происшествиями **уровня серьезности 3** .

После запуска мастера вы узнаете, как определить имя, источник контента и условия для типа результатов. С помощью представления списка можно определить приоритет типа результата.
  
### <a name="create-a-result-type-at-the-organization-level"></a>Создание типа результатов на уровне Организации

1. В [центре администрирования](https://admin.microsoft.com)перейдите к разделу **Настройка** > **службы поиска Microsoft**и выберите **Тип результата**.
1. Чтобы добавить **Тип результата**, нажмите кнопку **Добавить**. Чтобы изменить тип результата, выберите тип результата в соответствующем списке.

### <a name="create-a-results-type-at-the-site-level"></a>Создание типа результатов на уровне сайта

1. На сайте [SharePoint](http://sharepoint.com/) , на котором вы хотите создать тип результата, перейдите к разделу **Параметры**.
1. Выберите **сведения о сайте** , а затем **Просмотрите все параметры сайта**.
1. Найдите раздел Microsoft Search, а затем выберите **Настройка службы поиска Майкрософт для этого семейства веб-сайтов**.
1. В области навигации перейдите к **элементу пользовательский интерфейс**, а затем выберите вкладку **Тип результата** .
1. Чтобы добавить тип результата, нажмите кнопку **Добавить**.  
ИЛИ  
Чтобы изменить тип результата, выберите тип результата в списке.

### <a name="view-the-vertical-after-enabling"></a>Просмотр вертикальной оси после включения

После включения вертикальной установки может потребоваться некоторое время, прежде чем вы сможете просмотреть его.
Если вы не хотите дождаться его включения, вы можете добавить **качеклеар = true** в URL-адрес в [SharePoint](http://sharepoint.com/) и [Office](https://Office.com) для немедленного просмотра по вертикали.

## <a name="troubleshooting"></a>Устранение неполадок

Ниже перечислены распространенные проблемы, которые могут возникнуть, и действия по их устранению.

|Error  |Action  |
|---------|---------|
|Я вижу *неправильное* сообщение об ошибке на вертикальном. |   Для завершения настройки необходимы как вертикальные, так и типы результатов. Убедитесь, что вы создали оба для одного источника контента.      |
|Почему не отображается макет результатов, хотя он создан? | Использование типов результатов занимает несколько минут, так как эти параметры, как правило, кэшируются. Подождите несколько минут и повторите попытку.        |
|На странице "вертикальный текст" или "тип результата" не отображаются источники контента.     |      Убедитесь, что вы настроили соединители и индексированные данные.   |
