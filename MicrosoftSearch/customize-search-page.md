# <a name="customize-the-search-results-page"></a>Настройка страницы результатов поиска

Вы можете создавать вертикали поиска и типы результатов для настройки результатов поиска, которые пользователи видят при поиске в Microsoft [SharePoint,](https://sharepoint.com/) [Microsoft Office](https://office.com)и Поиске (Майкрософт) [в Bing.](https://bing.com) Вертикали упрощают для пользователей поиск сведений, которые у них есть разрешение на просмотр. Например, можно создать вертикаль поиска для маркетинговых данных анализа из сторонного программного обеспечения для пользователей в отделе маркетинга. Можно также определить типы результатов и настроить макет для этих данных.  

Вертикали и типы результатов можно создавать на этих уровнях:

- **На уровне** организации — при добавлении вертикали на уровне организации она отображается на странице результатов поиска, когда пользователи ведут поиск со своей стартовой страницы [SharePoint,](https://sharepoint.com/) [Office](https://office.com)или [Bing.](https://bing.com)
- **Уровень сайта—** например, вы можете позволить сотрудникам отдела обслуживания клиентов искать инциденты с уровнем серьезности *1* непосредственно на сайте SharePoint их отдела.

## <a name="search-verticals-explained"></a>Объясняются вертикали поиска

В верхней части страницы результатов Поиска (Майкрософт) есть строка вкладок. Это вертикали поиска. Вертикаль поиска отображает только результаты определенного типа или определенного контента. Примеры: **"Файлы"** или **"Новости".** По умолчанию Поиск (Майкрософт) отображает вертикали **"Все",** **"Люди",** **"Файлы",** **"Сайты"** и **"Новости".**  

Вы можете добавить вертикали поиска, релевантные для вашей организации. Они появятся на странице результатов поиска (Майкрософт) [в SharePoint,](https://sharepoint.com/) [Office](https://Office.com)и [Bing.](https://bing.com) Например, можно создать вертикаль для контента, связанного с маркетингом, и другую для продажи, в зависимости от типа информации, необходимой каждой группе. Вы можете добавить вертикали, чтобы показывать результаты только из контента, индексного через соединители.  

### <a name="multiple-connections-in-a-vertical"></a>Несколько подключений в вертикали

Вертикаль поиска теперь может использовать результаты из нескольких источников соединителя. Это обеспечивает большую гибкость при разработке страницы результатов поиска. Существующий административный опыт вертикальной настройки позволяет выбрать несколько подключений на шаге "Источник контента".
Если вы точно назначим как можно больше семантических меток, этот опыт будет улучшен. Вы можете добавлять семантические метки при определении схемы и их инициации.

[Ниже](configure-connector.md#step-5-assign-property-labels) данная информация о создании семантических меток и управлении ими.

### <a name="things-you-should-know"></a>Что необходимо знать

1. Подключение можно добавить в качестве источника контента только в рамках одной вертикали. Повторное использовать подключения по нескольким вертикалям запрещено.
2. Если требуется настроить запрос для вертикали поиска, в которую добавлено несколько источников подключений, для создания такого запроса следует использовать общие свойства источника.

## <a name="things-to-consider"></a>Важные факторы

Перед началом убедитесь, что соединители проиндексировали. Это может занять до 48 часов в зависимости от размера файла.

Вы не можете создать вертикаль для контента, расположенного в [SharePoint.](https://sharepoint.com/)

Существует три основных шага для добавления вертикали:

1. Создайте вертикаль. На этом этапе вы определяете имя вертикали, источник контента и область контента для поиска.
2. Определите, как будут выглядеть результаты для этой вертикали.  
3. Включить вертикаль (для отображения) со страницы вертикального списка.

## <a name="step-1-create-the-search-vertical"></a>ШАГ 1. Создание вертикали поиска

После запуска мастера вы сможете определить имя вертикали, источник контента и область контента для поиска. Вертикаль создается в отключеном состоянии. Вы ввяжем его позже.

Чтобы сузить область, можно использовать ограниченный набор языка запросов по ключевым словам [(KQL).](https://docs.microsoft.com/sharepoint/dev/general-development/keyword-query-language-kql-syntax-reference) На этой странице перечислены доступные свойства. Для создания KQL рекомендуется использовать текстовые ключевые слова и ограничения свойств с помощью boolean-операторов.

### <a name="create-a-vertical-at-the-organization-level"></a>Создание вертикали на уровне организации

Чтобы создать вертикаль в Поиске [(Майкрософт)](https://sharepoint.com/) в домашней учетной записи SharePoint, [Office](https://office.com)или [Bing,](https://bing.com)выполните следующие действия:

1. В Центре [администрирования Microsoft 365](https://admin.microsoft.com)перейдите в [**"Вертикали".**](https://admin.microsoft.com/Adminportal/Home#/MicrosoftSearch/verticals)
2. Выберите **"Добавить"** для начала работы.  

### <a name="create-a-vertical-at-the-site-level"></a>Создание вертикали на уровне сайта

1. На [сайте SharePoint,](https://sharepoint.com/) где нужна вертикаль, перейдите в **раздел "Параметры".**
2. Выберите **сведения о сайте,** а **затем просмотреть все параметры сайта.**
3. Наберем раздел **Поиска (Майкрософт)** и выберите "Настройка **Поиска (Майкрософт) для этого веб-сайта".**
4. В области навигации перейдите к **пользовательскому опыту** и выберите вкладку **"Вертикали".**
5. Чтобы добавить вертикаль, выберите **"Добавить".**
  Или, чтобы изменить вертикаль, выберите ее в списке.

Помните, что вертикали создаются в отключеном состоянии. Они должны быть включены, чтобы пользователи могли их видеть.

## <a name="step-2-create-the-result-types"></a>ШАГ 2. Создание типов результатов

Вы можете определить, как результаты отображаются в вертикали, разработать макет с помощью типов результатов. Макет результатов позволяет показывать важную информацию непосредственно в результатах поиска, поэтому пользователям не нужно выбирать каждый результат, чтобы узнать, нашли ли они то, что они ищут.

### <a name="default-search-result-layout"></a>Макет результатов поиска по умолчанию

Макет результатов поиска по умолчанию будет  показан для  контента соединителю, если метки и свойство контента были правильно соединялись с исходными свойствами во время настройки соединителю. Название **метки** является самой важной меткой. Настоятельно рекомендуется **иметь свойство,** назначенное этой метке, для использования макета результатов поиска по умолчанию.

### <a name="create-your-own-result-type"></a>Создание собственного типа результатов

Вы можете создать собственный макет результатов поиска и переопредить макет результатов поиска по умолчанию, создав **тип результата.** Тип результата поиска  это правило, благодаря которому результаты поиска разных типов отображаются различными способами. Оно состоит из следующих компонентов.

- **Одно или несколько условий** для сравнения каждого результата поиска, например источник контента результата поиска.  
- Макет **результатов для** результатов поиска, которые соответствуют условиям. Макет результатов управляет тем, как все результаты, которые соответствуют условиям, отображаются и ведут себя на странице результатов поиска.

**Если соответствующее сопоставление не сделано для отображения макета результатов поиска по умолчанию, необходимо создать по крайней мере один тип результатов для отображения результатов на вертикали.** Для каждой вертикали можно создать несколько типов результатов, что позволяет использовать различные макеты для различных типов результатов. Например, можно настроить инциденты серьезности *1* для более заметных цветов и шрифта большего размера по сравнению с инцидентами со степенью *серьезности 3.*

После запуска мастера вы сможете определить имя, источник контента и условия для типа результатов. Можно определить приоритет типа результата из представления списка.
  
### <a name="create-a-result-type-at-the-organization-level"></a>Создание типа результата на уровне организации

1. В Центре [администрирования Microsoft 365](https://admin.microsoft.com)перейдите к [**типам результатов.**](https://admin.microsoft.com/Adminportal/Home#/MicrosoftSearch/resulttypes)
2. Чтобы добавить тип **результата, выберите** **"Добавить".** Чтобы изменить тип результата, выберите его в соответствующем списке.

### <a name="create-a-results-type-at-the-site-level"></a>Создание типа результатов на уровне сайта

1. На [сайте SharePoint,](https://sharepoint.com/) где нужно создать тип результата, перейдите в раздел **"Параметры".**
2. Выберите **сведения о сайте,** а **затем просмотреть все параметры сайта.**
3. Наберем раздел Поиска (Майкрософт) и выберите "Настройка **Поиска (Майкрософт) для этого веб-сайта".**
4. В области навигации перейдите к **пользовательскому опыту** и выберите вкладку **"Тип результата".**
5. Чтобы добавить тип результата, выберите **"Добавить".**  Или, чтобы изменить тип результата, выберите тип результата в списке.

## <a name="step-3-view-the-vertical-after-its-enabled"></a>ШАГ 3. Просмотр вертикали после включения

После этого может потребоваться некоторое время, прежде чем вы сможете просмотреть вертикаль. Если вы не хотите ждать его включения, можно сразу же применить **cacheClear=true** к URL-адресу [в SharePoint](https://sharepoint.com/) и [Office,](https://office.com) чтобы сразу просмотреть вертикаль. Для [Bing](https://bing.com),&**features=uncachedVerticals** к URL-адресу вертикали работы для немедленного просмотра вертикали.

## <a name="troubleshooting"></a>Устранение неполадок

Вот список распространенных проблем, с которыми вы можете столкнуться, и действий по их устранению.

|Ошибка  |Действие  |
|---------|---------|
| Я вижу сообщение об ошибке "Что-то пошло не так" на вертикали. | Для завершения настройки необходимы как вертикаль, так и типы результатов. Убедитесь, что вы создали оба источника контента. |
| Я не вижу свой макет результатов, хотя я создал его. | Это занимает несколько минут, так как эти параметры обычно кэшются. Подождите несколько минут и попробуйте еще раз.        |
| Я не вижу источники контента на странице вертикали или типа результата. | Убедитесь, что вы настроили соединители и индексировали данные.   |
