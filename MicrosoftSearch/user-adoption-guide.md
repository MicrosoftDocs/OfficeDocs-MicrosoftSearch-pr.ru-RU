# <a name="user-adoption-guide"></a>Руководство по внедрению среди пользователей

Эти сведения предназначены для администраторов Microsoft Search, руководителей управления изменениями и владельцев предприятий.
  
Чтобы выполнить развертывание службы поиска Майкрософт в Организации и обеспечить их успешную работу, рекомендуется выполнить этот процесс в четырех стадиях.
  
## <a name="stage-1-stakeholders"></a>Этап 1: заинтересованные стороны

Чтобы обеспечить бесперебойную расразвертыванию и улучшить внедрение Microsoft Search, определите людей, которые могут помочь в поддержке и работе этого процесса.
  
В таблице показана иерархия команд и сведения о каждом участнике. 
  
 **Роли и обязанности команды по внедрению пользователей**
  
|**Role**|**Несут**|**Department**|
|:-----|:-----|:-----|
|Ответственный спонсор  <br/> |Общие сведения о высоком уровне концепции и значении службы поиска Майкрософт для Организации  <br/> Предоставление поддержки руководства для ИТ-бизнеса и бизнес-групп  <br/> Выделение ресурсов  <br/> Непосредственное взаимодействие с сотрудниками  <br/> |Руководящий лидер  <br/> |
|Владелец бизнеса  <br/> |Определение сценариев продукта и пользователей  <br/> Определение метрик успешности для отделов и Организации  <br/> |Любые  <br/> |
|Группа управления изменениями  <br/> |Обеспечение легкого знакомства со значением продукта  <br/> Создание плана для развертывания  <br/> |IT  <br/> |

## <a name="stage-2-scenarios"></a>Этап 2: сценарии

Чтобы обеспечить внедрение пользователей, важно определить и определить сценарии конечных пользователей. Они должны включать реальные примеры того, как пользователи могут использовать Microsoft Search для поиска необходимой информации, ответов на вопросы и выполнения задач быстрее. Чтобы измерить успешность развертывания и внедрения пользователей, также важно определить метрики, которые вы будете использовать перед началом работы.
  
 **Общие сведения о том, что нужны пользователям**
  
Когда речь идет о поиске информации и получении ответов, знание потребностей и точек зрения пользователя — ключ. Для получения этих данных можно использовать количественные методы, например данные из Office 365 и использование поиска SharePoint, а также качественные методы, например опросы и группы фокуса пользователя. Это поможет определить, когда и как Microsoft Search может предоставить значение.
  
Для различных ролей и Teams в Организации следует рассмотреть следующие возможности:
  
- При поиске сведений существующие пользователи замоменты.
    
- Часто используемые приложения, инструменты, сайты и другие источники данных.
    
- Какие пользователи хотели бы видеть, когда заходит Поиск информации.
    
- Преимущества Microsoft Search, применяемой к вашей организации. Более подробную информацию можно узнать в [статье Почему Microsoft Search](why-microsoft-search.md).
    
Выберите сценарии пользователей, которые будут иметь наибольшее положительное воздействие на ваших пользователей.
  
 **Определение метрик успешности**
  
Чтобы обеспечить соответствие целям развертывания и оказать положительное воздействие на вашу организацию, прежде чем начать, важно определить основные показатели успеха. Придерживайтесь простых и измеримых целей и регулярно оцените производительность, чтобы при необходимости можно было быстро реагировать на них.
  
1. **Управление изменениями:** Определите, как будет измеряться служба поиска Майкрософт, а также успешность ее связи. 
    
2. **Использование:** Вы можете использовать данные из панели мониторинга Insights, в том числе ежедневные и ежемесячные активные пользователи (ДАУ и Майкрософт), типы контента, которые получают наибольшее число просмотров, и ключевые слова верхнего поиска для измерения использования со временем. Для получения дополнительных сведений см [](get-insights.md).
    
3. **Удовлетворенность пользователей:** Для определения базового плана используйте данные, собранные из опросов и групп фокусов пользователей. Нетто-удовлетворенность (НСАТ) и оценки переоценки серверов в сети могут также предоставлять полезные измерения. Независимо от используемого метода, его следует поддерживать единообразно, чтобы можно было видеть влияние на время. 
    
4. **Другие измерения:** Используйте исторические и/или текущие данные для запросов в службу поддержки и время, затрачиваемое на поиск. 
    
## <a name="stage-3-readiness"></a>Этап 3: готовность

Время, в течение которого подготавливается к развертыванию Microsoft Search, приводит к положительному появлению на предприятии. Корпорация Майкрософт предоставляет <a href="https://resources.techcommunity.microsoft.com/microsoft-search-in-bing-adoption-kit/">набор для внедрения</a> , который содержит рекомендации по планированию, настраиваемые объявления электронной почты, шаблоны опросов, руководящие презентации и многое другое. Кроме того, в <a href="https://resources.techcommunity.microsoft.com/microsoft-search/#adoption">центре ресурсов поиска Майкрософт</a> можно найти другие материалы по обучению и освоению, разработанные для достижения успеха при пробной отучении Microsoft Search.
  
 **Готовность продукта и контента**
  
Администраторы непосредственно влияют на возможности поиска для конечных пользователей и определяют, как Microsoft Search ищет организацию. Сюда относится выбор типов результатов, которые должны отображаться для пользователей. Эти результаты могут включать ссылки на документы, сайты, людей, группы, расположения, беседы и другие ресурсы рабочего места, а также известные как закладки.
  
Дополнительные сведения: [Настройка службы поиска Майкрософт](setup-microsoft-search.md).
  
Редакторы являются экспертами, которые помогают управлять закладками и Q&в соответствии с их областями. Редакторы должны создавать содержимое, которое решает пользовательские сценарии, которые вы определили ранее.
  
Для получения дополнительных сведений ознакомьтесь со статьей [Планирование контента](plan-your-content.md).
  
 **Вход**
  
Пользователи службы поиска Майкрософт должны войти в службу Bing с помощью рабочей или учебной учетной записи. Чтобы обеспечить максимально возможный вход для пользователей, выполните следующие действия.
  
- Убедитесь, что конфигурации Azure Active Directory настроены правильно, несколько пользователей пытаются войти в систему. Более подробную информацию можно узнать [в статье тестирование единого входа](test-single-sign-on.md).
    
- Задать параметры браузера по умолчанию для устройств, управляемых компанией, в Bing. Дополнительные сведения можно найти в разделе [Установка браузера по умолчанию](set-default-browser.md).
    
- Разверните расширения поиска для пограничного пользователя и пользователя Chrome с помощью групповой политики. Для получения дополнительных сведений обратитесь к разделу [инструменты портала администрирования](admin-portal-tools.md).
    
 **Черновики для общения**
  
Используйте реальные сценарии, чтобы сообщать пользователям о поиске в Майкрософт и о том, как они помогают экономить время. Группа управления изменениями поможет вам создать стратегию развертывания и подготовить материалы и кампании для обеспечения готовности пользователей к работе.
  
Начните планировать связь и события для Организации:
  
- Плакаты, афиши или леафлетс, размещенные в выразительном расположении
    
- Демонстрации и стенды для собственных продуктов
    
- Интерактивные события для локальных и удаленных команд
    
- Попросите ведущего спонсора создать собрание или отправить сообщение электронной почты с оповещением.
    
- Совместное использование справочной документации по поиску в Майкрософт
    
Все коммуникации должны проинструктировать пользователей о том, как они могут войти в Bing, чтобы найти результаты работы.
  
 **Участие в группах партнеров**
  
Как неотъемлемой частью Microsoft 365, служба поиска Майкрософт соответствует рекомендациям по GDPR и уровню с. В зависимости от вашей организации может потребоваться предоставить общий доступ к сведениям о поиске Майкрософт для групп конфиденциальности и безопасности. Более подробную информацию можно узнать в [статье Security for Microsoft Search](security.md).
  
Убедитесь, что ваша служба технической поддержки знает о возможностях и функциях Microsoft Search, а также требованиях пользователей к входу, чтобы они могли более эффективно поддерживать пользователей. Дополнительные сведения см. [](use/about-microsoft-search.md) [](faqs.md)
  
 **Оценка с помощью пилотных пользователей**
  
Начните с небольшой группы пользователей, чтобы оценить влияние службы поиска Майкрософт и собрать начальную обратную связь. Это дает время на изменение контента, обновление связи и настройку ожидания для широкого развертывания. Поиск "Опытные пользователи" или ранние пользователи имеют самые эффективные кандидаты.
  
Чтобы помочь пользователям выполнять поиск пилотных версий:
  
- Сотрудничество с инвестором в организации
    
- Определение группы ранних последователей
    
- Определение групп, выполняющих большое количество операций корпоративного поиска
    
- Определение пользователей в других географических расположениях, в том числе в других странах
    
Попросите ваших пилотных пользователей ввести данные с помощью параметров отзывов в Microsoft Search или с помощью простых опросов.
  
Вопросы опроса: 
  
- Вы знаете, как войти и использовать службу поиска Майкрософт?
    
- Работает ли он должным образом?
    
- Какие функции вам понравилось?
    
- Удалось найти результаты работы?
    
- Полезными результаты работы?
    
- Являются сообщениями о том, что служба поиска Майкрософт понятна и понятна?
    
Это отличное время для начала измерения эффективности метрик, определенных на этапе 2.
  
При запуске пилотного этапа рекомендуется предоставить краткий обзор Microsoft Search с использованием более широкой организации.
  
## <a name="stage-4-roll-out"></a>Этап 4: развертывание

Каждая организация рассчитает, что продукт полагается по-разному. Вот несколько моментов, которые следует учитывать.
  
 **Широкое развертывание**
  
Когда вы будете готовы, разрешите всем пользователям организации доступ к Microsoft Search или поэтапный подход к более медленному развертыванию. 
  
 **Создание сведений о состоянии**
  
Создание в организации сведений о поиске в Майкрософт:
  
- Прикрепите пилотных пользователей к обеспечению одноранговой поддержки и опыта.
    
- Следуйте указаниям по связи и событиям, запланированным в шаге 3.
    
 **Непрерывная связь**
  
В течение первых нескольких недель и месяцев Храните ритм взаимодействия для поощрений в Microsoft Search. Это позволяет пользователям получать уведомления и привлекать новых пользователей. Каждая связь должна объяснить, почему Microsoft Search может быть полезен для работы.
  
Отправка текущих коммуникаций, выделяющих функции и результаты поиска, а также истории успеха пользователей.
  
 **Сбор отзывов и улучшений**
  
Поддержка пользователей и информирование пользователей о них, прослушивая их и действуя над их отзывами.
  
Используйте панель мониторинга аналитики, чтобы узнать, как выполняется продукт и его содержимое. Используйте эту информацию для обновления закладок, Q&как и других коммуникаций. Кроме того, вы можете сделать следующее:
  
1. Повышение качества контента: заголовки, описания и URL-адреса, Оптимизация ключевых слов, удаление устаревших сведений и добавление новых сведений
    
2. Определение наиболее часто используемых функций (или нередко)
    
3. Проведение опросов, групп фокусов и сеансов обратной связи, чтобы понять, что пользователи думают об поиске в Майкрософт, увеличивают внедрение и выдают области улучшения
    
Образование и обратная связь — это итеративный процесс. Может потребоваться внести изменения в контент и коммуникации, так как ваши пользователи выполняют поиск Microsoft Search.
